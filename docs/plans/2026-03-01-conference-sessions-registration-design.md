## Section 1: Sessions Panel (Left)

### Data
- Source: `sessions.md` — 38 sessions, each with: title, organiser(s), general topic, abstract text
- Implementation: Hardcoded typed array in `src/data/sessions.ts` (generated by one-off parser script, not runtime parsed)
- Type: `{ id: number; title: string; organizers: string; topic: string; abstract: string }`

### UI
- `src/components/conference/session-list.tsx` — client component
- Shows "The call for sessions has now closed. The following thematic sessions have been accepted."
- Numbered list: `1. [Title]` … `38. [Title]` — each is a clickable button
- Click opens **Radix Dialog** (`@radix-ui/react-dialog` already installed) showing:
  - Session number + title (large heading)
  - **Organizers:** value
  - **General topic:** value
  - **Abstract:** full text (scrollable if long)
- Dialog closes on backdrop click or Escape

## Section 2: Abstracts Panel (Right)

### Layout
Three stacked cards (flex-col gap), one below the other:
1. Step 1 — Register (`ClipboardList` icon)
2. Step 2 — Submit Abstract (`FileText` icon)
3. Step 3 — Upload Payment Receipt (`Receipt` icon)

Each card: icon + step label + title + description + button that opens its dialog.

### Step 1: Registration Dialog
**File:** `src/components/conference/registration-dialog.tsx`
**Fields:**
- First Name, Last Name (required)
- Email (required) + Verify button (sends test email to confirm address)
- Affiliation (required)
- Country (required)
- Registration Type (required) — select with 4 options showing both early/late bird prices:
  - Regular (Early €60 / Late €80)
  - HGS Member (Early €40 / Late €50)
  - Student (Early €20 / Late €30)
  - HGS Student Member (Early €10 / Late €15)
- Abstract Intent — oral / poster / not submitting
- Dietary Requirements — none / vegetarian / vegan / kosher / gluten-free / other
- Dietary Other (shown if "other" selected)
- Special Requirements (optional textarea)

**Logic:**
- Duplicate email check against Supabase `registrations` table before insert
- On success: insert to Supabase, fire-and-forget POST to `/api/send-email` (type: "registration")
- Success state: green checkmark + confirmation message

### Step 2: Abstract Submission Dialog
**File:** `src/components/conference/abstract-dialog.tsx`
**Fields:**
- Registration Email (required) — looked up against `registrations` to prefill name/affiliation
- Abstract Title (required)
- Session (required) — dropdown of all 38 sessions by number + title
- Co-authors (optional)
- Presentation Type — oral / poster
- Abstract Text (textarea, max 500 words, live word counter)
- OR file upload (.docx / .pdf, max 10MB)

**Logic:**
- Must be registered (email lookup)
- Duplicate abstract check
- File upload to Supabase storage bucket `abstracts`
- On success: insert to `abstracts` table, fire-and-forget email (type: "abstract")

### Step 3: Payment Receipt Dialog
**File:** `src/components/conference/payment-dialog.tsx`
**Fields:**
- Registration Email (required)
- Receipt Type — Conference Payment / HGS Membership Payment
- File upload (PDF/JPG/PNG, max 10MB, required)
- Notes (optional textarea)

**Logic:**
- File upload to Supabase storage bucket `payment-receipts`
- On success: insert to `payment_receipts` table, fire-and-forget email (type: "receipt")

## Section 4: Supabase Schema (SQL to run)

Three tables + two storage buckets, all with RLS enabled.

### `registrations`
```sql
CREATE TABLE IF NOT EXISTS public.registrations (
  id                   UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at           TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  updated_at           TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  first_name           TEXT        NOT NULL,
  last_name            TEXT        NOT NULL,
  email                TEXT        NOT NULL UNIQUE,
  affiliation          TEXT        NOT NULL,
  country              TEXT        NOT NULL,
  registration_type    TEXT        NOT NULL
    CHECK (registration_type IN ('regular', 'hgs_member', 'student', 'hgs_student')),
  abstract_intent      TEXT        NOT NULL DEFAULT 'none'
    CHECK (abstract_intent IN ('oral', 'poster', 'none')),
  dietary              TEXT        NOT NULL DEFAULT 'none'
    CHECK (dietary IN ('none', 'vegetarian', 'vegan', 'kosher', 'gluten_free', 'other')),
  dietary_other        TEXT,
  special_requirements TEXT        NOT NULL DEFAULT '',
  payment_confirmed    BOOLEAN     NOT NULL DEFAULT FALSE
);
```

### `abstracts`
```sql
CREATE TABLE IF NOT EXISTS public.abstracts (
  id                UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at        TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  first_name        TEXT        NOT NULL,
  last_name         TEXT        NOT NULL,
  email             TEXT        NOT NULL,
  affiliation       TEXT        NOT NULL,
  session_id        INTEGER,
  title             TEXT        NOT NULL,
  co_authors        TEXT        NOT NULL DEFAULT '',
  abstract_text     TEXT,
  file_path         TEXT,
  presentation_type TEXT        NOT NULL
    CHECK (presentation_type IN ('oral', 'poster')),
  CONSTRAINT abstracts_content_check
    CHECK (abstract_text IS NOT NULL OR file_path IS NOT NULL)
);
```

### `payment_receipts`
```sql
CREATE TABLE IF NOT EXISTS public.payment_receipts (
  id           UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at   TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  email        TEXT        NOT NULL,
  receipt_type TEXT        NOT NULL
    CHECK (receipt_type IN ('conference', 'hgs_membership')),
  file_path    TEXT        NOT NULL,
  notes        TEXT        NOT NULL DEFAULT ''
);
```

### Storage buckets
- `abstracts` — private, 10MB, .docx/.doc/.pdf
- `payment-receipts` — private, 10MB, PDF/JPG/PNG/WEBP

### RLS policies
- All three tables: anon INSERT + anon SELECT (same pattern as DenuChange)
- Both buckets: anon INSERT
